# 项目结构说明

当前仓库已经按“现役代码、安装打包、文档资料、旧版代码”分开整理。

## 现在真正使用的内容

- `src/`
  - 当前正式版本源码
  - 主程序在 `src/Monitor.App/`
- `installer/`
  - 安装脚本和安装打包脚本
- `tools/`
  - 采样、诊断、辅助脚本
- `artifacts/`
  - 发布产物、安装包、构建输出
- `cloudflared.exe`
  - 公网发布依赖
- `README.md`
- `PRODUCT.md`
- `DESIGN.md`

## 文档资料

- `docs/plans/`
  - 任务计划、修复计划、实现说明
- `docs/design/`
  - 设计预览、界面截图、审查图
- `docs/chat/`
  - 过程资料和对话留档

## 旧版代码

- `legacy/python-monitor/`
  - 旧版 Python 项目
  - 已把旧数据库、旧配置、旧日志一起归档到这里

## 目前顶层保留原则

顶层只保留下面几类内容：

- 正在使用的代码
- 安装和发布所需文件
- 必要的总说明文件
- 明确分类后的文档目录
- 明确分类后的旧版归档目录

## 下一步建议

下一步不要直接删文件，先做一次“可删除清单”确认：

1. 哪些旧版文件只是留档，不再需要
2. 哪些设计截图已经重复
3. 哪些历史计划文件可以合并
4. 哪些构建产物可以清空后重新生成
